Implementations

Rendezvous Java language interface offers two implementations. Table 1 summarizes and compares them. To use a specific implementation, see Tibrv.open() .

Table 1 Implementations 
Aspect
Native (JNI)
Java
Description
A thin Java layer serves as a programming interface to an underlying C implementation.
Implemented in Java.
Primary Runtime Environment
Independent application or browser applet.
Downloadable browser applet.
Scope of Functionality
Maximal coverage of C functionality. Exception: I/O events.
Covers core functionality of Rendezvous. Does not include fault tolerance, certified delivery, nor secure daemons.
Speed
Generally faster than the Java implementation.
Generally not as fast as the native C implementation.
Transport Scope
rvd, rva, and intra-process transports.
Only rva and intra-process transports. Does not support rvd transports.
Timer Granularity
Supports fine-resolution timers (within operating system limitations).
Timer granularity is limited by JVM to 10 milliseconds.
Library Requirements
Must load the Rendezvous JNI library.
Does not require the Rendezvous JNI library.
