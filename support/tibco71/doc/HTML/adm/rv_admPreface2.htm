Manual Organization

This book begins with important information for system administrators:

  • Chapter 1, Do This First—Administrator’s Checklist
  • Chapter 2, Licensing Information

The third chapter describes several details upon which programmers and administrators must agree for correct program operation:

  • Chapter 3, Network Details

The next several chapters describe Rendezvous components that run as executable processes. Administrators must ensure correct set-up and operation of these components:

  • Chapter 4, Rendezvous Daemon (rvd)
  • Chapter 5, Routing Daemon (rvrd)
  • Chapter 6, Secure Daemons (rvsd and rvsrd)
  • Chapter 7, Relay Agent
  • Chapter 8, Rendezvous Agent (rva)
  • Chapter 9, Current Value Cache

Two chapters describe utilities for measuring overall system capacity and performance, and for diagnosing network problems.

  • Chapter 10, Performance Assessment (rvperf)
  • Chapter 11, Protocol Monitor (rvtrace)

Many administrators use the Perl programming language for system administration tasks. A brief chapter describes the Rendezvous Perl API:

  • Chapter 12, Perl 5 Interface

Additional administrative tasks apply when distributed systems use advanced features of Rendezvous software:

  • Chapter 13, Certified Message Delivery
  • Chapter 14, Fault Tolerance
  • Chapter 15, Distributed Queues

To register Rendezvous software with the Microsoft Windows operating system, use these utilities:

  • Appendix A, Windows Services

Architectural changes in release 6 lead to several important changes for administrators:

  • Appendix B, Changes in Release 6
