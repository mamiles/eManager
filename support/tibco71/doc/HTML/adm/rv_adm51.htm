Daemon Parameters

This page lets you configure parameters that affect overall daemon security.

To display this page, click Daemon Parameters in the left margin of any page of the rvrd browser administration interface.

Figure 23 rvrd Daemon Parameters Page

Administrator Password

Only authorized personnel have access to routing daemons.

When administrator identification information is not set, anyone who can connect to the browser administration interface can examine and reconfigure the daemon. This arrangement can be useful during initial configuration and testing phases. However, during regular operation we recommend limiting access.

Once administrator identification information is registered, the browser administration interface is locked against unauthorized access. The daemon prompts the administrator to prove identity by typing a name and password. After providing proper identification, the authorized administrator has complete access. If the administrator does not provide proper identification, the browser displays the General Information page and continues to prompt for a correct name and password.

Successful administrator identification remains in effect for the duration of the browser process. Merely closing the browser window does not end administrator access—to guard against intruders you must terminate the browser process (all its windows).

 

Item
Description
Name
Type a name string.
Password
Type a password string.

Type the password again.

Specify a name and password, then click this button to lock the interface.
Delete Administrator
Click this button to delete administrator identification information and unlock the interface. (This button appears only when administrator identification has been set.)

Logging

This panel configures the kind of routing activity that the routing daemon routinely outputs to its log file.

(Sheet 1 of 2)

Item
Description
Connections
Log connection activity whenever this routing daemon establishes or closes a connection to a neighbor.
Subject Interest
Log all subscription requests (notification of listening) that this routing daemon sends to its neighbors or receives from its neighbors.
Subject Data
Log all messages that this routing daemon forwards to its neighbors or receives from its neighbors.

To configure the destination of logging output, see Routing Daemon Logging.
