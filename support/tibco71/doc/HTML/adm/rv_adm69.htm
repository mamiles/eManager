Daemon Parameters

This page lets you configure parameters that affect overall daemon security.

To display this page, click Daemon Parameters in the left margin of any page of the secure daemon browser administration interface.

For rvsd, this page contains two panels—Administrator Password and Default Network and Service. For rvsrd, this page adds a third panel for logging parameters; see Logging.

Figure 36 rvsd Daemon Parameters Page

Administrator Password

Only authorized personnel have access to secure daemons.

When administrator identification information is not set, anyone who can connect to the browser administration interface can examine and reconfigure the daemon. This arrangement can be useful during initial configuration and testing phases. However, during regular operation we recommend limiting access.

Once administrator identification information is registered, the browser administration interface is locked against unauthorized access. The secure daemon prompts the administrator to prove identity by typing a name and password. After providing proper identification, the authorized administrator has complete access. If the administrator does not provide proper identification, the browser displays the General Information page and continues to prompt for a correct name and password.

Successful administrator identification remains in effect for the duration of the browser process. Merely closing the browser window does not end administrator access—to guard against intruders you must terminate the browser process (all its windows).

Default Network and Service

Secure daemons and non-secure daemons behave differently when a client transport specifies a default value for its network or service parameter. Non-secure daemons use external defaults; see Specifying the UDP or PGM Service and Constructing the Network Parameter. In contrast, secure daemons use internal defaults—which you can configure using this panel. (For background information, see Network and Service Authorization.)
