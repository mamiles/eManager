TibrvListener

Class

Purpose

Listen for inbound messages.

Remarks

A listener object continues listening for messages until the program destroys or deletes it.

Destroying the queue or transport of a listener automatically destroys the listener as well, but does not delete the COM listener object.

Activation and Dispatch

Inbound messages on the transport that match the subject trigger the event.

TibrvListener.create creates the internal structure of a listener object, and activates event processing—that is, it begins listening for all inbound messages with matching subjects. When a message arrives, Rendezvous software places the listener object and message on its queue. Dispatch removes the listener object from the queue, and runs the callback method to process the message. (To stop receiving inbound messages on the subject, destroy or delete the listener object; this action cancels all messages already queued for the listener; see also TibrvListener.destroy .)
